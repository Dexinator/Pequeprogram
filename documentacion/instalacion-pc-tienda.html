Entrepeques · POS modernizado

Guía de instalación en la PC de tienda

Sistema de tickets, etiquetas y escáner · Windows
En esta guía:
  1. Lo que necesitas tener listo
  2. Instalar drivers de las impresoras
  3. Instalar el "Print Bridge" (servicio local de impresión)
  4. Configurar y arrancar el servicio
  5. Aceptar el certificado en el navegador
  6. Probar una venta y una etiqueta
  7. Si algo falla

1. Lo que necesitas tener listo

  • La PC de tienda con Windows y conexión a internet.
  • Cuenta con permisos de administrador (para instalar drivers y un servicio del sistema).
  • La impresora Epson TM-T88IV conectada por USB y prendida.
  • La impresora Zebra GC420t conectada por USB y prendida, con rollos de etiquetas 49 × 51 mm y ribbon cargados.
  • El escáner conectado por USB.

2. Instalar drivers de las impresoras

Epson TM-T88IV

  1. Abre epson.com.mx → Soporte → Buscar "TM-T88IV".
  2. Descarga el "Advanced Printer Driver (APD)" para Windows.
  3. Ejecuta el instalador siguiendo los pasos por defecto.
  4. Cuando termine, en Windows abre Configuración → Dispositivos → Impresoras y escáneres. Debe aparecer una entrada parecida a "EPSON TM-T88IV Receipt".

Zebra GC420t

  1. Abre zebra.com → Soporte → Buscar "GC420t".
  2. Descarga el driver "ZDesigner" para Windows.
  3. Instálalo y reinicia si te lo pide.
  4. En "Impresoras y escáneres" debe aparecer una entrada parecida a "ZDesigner GC420t".
Importante: anota en una hoja el nombre exacto que muestra Windows para cada impresora — incluyendo mayúsculas y espacios. Lo vas a usar en el paso 4. Para verlo abre PowerShell y escribe: 
Get-Printer | Select Name

3. Instalar el "Print Bridge"

El Print Bridge es un mini-programa que corre en esta PC y conecta el sistema de POS con las impresoras USB. Sin él, el POS no puede imprimir porque está en la nube.

Instalar Node.js (si no está)

  1. Abre nodejs.org y descarga la versión LTS (botón verde de la izquierda).
  2. Ejecuta el instalador con los valores por defecto.
  3. Para confirmar: abre PowerShell y escribe: 
    node --version
npm --version
    Deben mostrar números (Node v20.x o mayor).

Bajar el código del Print Bridge

  1. Abre PowerShell como administrador (botón derecho → "Ejecutar como administrador").
  2. Posiciónate donde quieras guardar el código, por ejemplo: 
    cd C:\
mkdir Entrepeques
cd Entrepeques
  3. Clona el repositorio (si te pide credenciales de GitHub, usa las del equipo): 
    git clone https://github.com/Dexinator/Pequeprogram.git
cd Pequeprogram\apps\print-bridge
  4. Instala dependencias y compila: 
    npm install
npm run build

4. Configurar y arrancar el servicio

Crear el archivo de configuración

  1. Aún en la carpeta apps\print-bridge, copia el ejemplo: 
    copy .env.example .env
  2. Abre .env con el Bloc de notas: 
    notepad .env
  3. Cambia estos valores (los demás los dejas como están): 
    ENABLE_HTTPS=true
RENDER_MODE=live
EPSON_PRINTER_NAME=NOMBRE EXACTO DE TU EPSON
ZEBRA_PRINTER_NAME=NOMBRE EXACTO DE TU ZEBRA
LABEL_WIDTH_DOTS=392
LABEL_HEIGHT_DOTS=408
  4. Guarda y cierra el archivo.
Los nombres de impresora son case-sensitive: tienen que coincidir exactamente con lo que vimos en Get-Printer, incluyendo mayúsculas y espacios.

Registrar como servicio de Windows

  1. En la misma PowerShell de administrador, ejecuta: 
    npm run service:install
  2. Verás mensajes confirmando que el servicio "EntrepequesPrintBridge" se instaló y arrancó.
  3. Para verificar que está vivo, abre tu navegador y entra a:
    http://localhost:9100/health
    Debe contestar con un mensaje en JSON que diga "ok": true.
A partir de ahora, el servicio arranca solo cada vez que prendas la PC y se reinicia si algo lo tira. No tienes que abrir consolas nunca más.

5. Aceptar el certificado en el navegador (una sola vez)

El sistema de POS vive en la web (Vercel) y por seguridad solo puede hablar con el Print Bridge si la conexión es HTTPS. La primera vez Windows no confía en el certificado del bridge porque es autogenerado, hay que aceptarlo manualmente.

  1. Abre el navegador que vas a usar para vender (Chrome, Edge o el que prefieras).
  2. Entra a:
    https://localhost:9443/health
  3. El navegador mostrará una advertencia de "Conexión no privada" o similar. Click en Configuración avanzada → Continuar de todos modos (el texto exacto cambia según navegador).
  4. Debe aparecer el mismo JSON de antes con "ok": true.
  5. Listo, ya no volverá a preguntar.
Repite este paso en cada navegador que vayan a usar para el POS.

6. Probar una venta y una etiqueta

  1. Entra al POS en la URL que te compartimos (Vercel staging o producción).
  2. Inicia sesión con tu usuario.
  3. Probar ticket: haz una venta de prueba (puede ser un producto cualquiera, en efectivo). Al cerrar la venta, presiona Imprimir Ticket. La Epson debe imprimir el ticket completo con el código de barras al pie.
  4. Probar etiqueta: entra al módulo de Inventario, busca cualquier producto y presiona Imprimir etiqueta. La Zebra debe imprimir la etiqueta con marca, talla, precio grande y código de barras.
  5. Probar escáner: en el módulo de Ventas, escanea una etiqueta recién impresa. El producto debe aparecer automáticamente en el carrito.

7. Si algo falla

Síntoma Qué probar
El botón "Imprimir Ticket" dice "No se pudo conectar al servicio de impresión local". 1) Abre https://localhost:9443/health en el navegador. Si no carga, el servicio se detuvo. Reinicia la PC para que arranque otra vez.
2) Si carga pero el navegador marca cert no confiable, repite el paso 5.
El ticket sale con caracteres raros (acentos, ñ). Verifica que en "Impresoras y escáneres" la Epson esté configurada en modo raw, no como "impresora genérica de texto".
La etiqueta sale corrida, desfasada o con cosas cortadas. La medida del rollo no coincide con la configuración. Edita el .env: cambia LABEL_WIDTH_DOTS y LABEL_HEIGHT_DOTS al nuevo tamaño (mm × 8). Luego reinicia el servicio desde "Servicios" de Windows: busca EntrepequesPrintBridge, click derecho, Reiniciar.
Error: OpenPrinter falló para '...'. El nombre de la impresora en .env no coincide con el de Windows. Revisa con Get-Printer | Select Name en PowerShell y corrige.
El escáner no agrega productos al escanear. 1) Verifica que el campo de búsqueda esté en foco. 2) Asegúrate de que el escáner esté configurado en layout Spain keyboard (en el manual del escáner busca ese código y escanéalo).
